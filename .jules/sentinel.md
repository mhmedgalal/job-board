## 2024-05-18 - Fix Open Mail Relay in contact form
**Vulnerability:** The application was vulnerable to Open Mail Relay in the contact form (`contact/views.py`). The `send_mail` function blindly used the user-provided `email` field from the POST request as the sole recipient. An attacker could exploit this by submitting any target email address and spam message, using the server to send arbitrary, untraceable spam/phishing emails on their behalf, potentially causing the server's IP to be blacklisted.
**Learning:** Never pass untrusted user input directly to the recipient field of email sending functions, especially in contact forms. The recipient should generally be a trusted, pre-configured address (e.g., admin).
**Prevention:** Hardcode the target recipient to the administrator's email address or a trusted fallback (`settings.DEFAULT_FROM_EMAIL`). Include the user's provided email address within the message body or as a Reply-To header, rather than allowing it to dictate the email's destination.
