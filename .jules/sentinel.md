## 2026-04-18 - Open Mail Relay in Contact Form
**Vulnerability:** The contact form's backend (`contact/views.py`) took an email address from the user request and used it as the recipient in the Django `send_mail` function. This created an open mail relay vulnerability, where attackers could use the application's infrastructure to send unauthorized emails to any destination address, potentially enabling spam or phishing campaigns.
**Learning:** Never trust user input for the recipient address when sending automated emails. When a form is intended to contact the site administrator, the recipient address should be statically defined in the configuration (e.g., `settings.DEFAULT_FROM_EMAIL`), and the user's provided email should only be included within the email body for reply purposes.
**Prevention:** Always hardcode or use environment variables for recipient email addresses in contact forms. Sanitize and validate all user input before inclusion in email bodies or headers. Ensure the `send_mail` function uses authorized and expected recipients.
