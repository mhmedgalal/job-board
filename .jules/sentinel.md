## 2025-03-26 - Insecure File Upload via Content-Type Spoofing
**Vulnerability:** The `upload_avatar` API endpoint in `accounts/view_api.py` relied solely on checking the `Content-Type` header (e.g., `image/jpeg`) to validate uploaded avatar images. An attacker could easily spoof this header while uploading a malicious executable file (like `shell.php`), leading to a critical remote code execution (RCE) vulnerability if the media directory executes scripts.
**Learning:** Checking the `Content-Type` header is insufficient for verifying the safety of file uploads, as this header is controlled by the client. Application logic must securely validate the actual file contents or, at a minimum, ensure the file extension matches a safe whitelist.
**Prevention:** Always validate uploaded files against a strict whitelist of safe file extensions. Avoid trusting client-provided metadata like `Content-Type`. If possible, process and re-encode images to ensure they do not contain hidden malicious payloads.
