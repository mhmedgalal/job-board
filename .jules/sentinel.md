## 2024-05-18 - Open Email Relay via Contact Form
**Vulnerability:** The contact form `contact/views.py` used the user-provided `email` input directly as the recipient address in Django's `send_mail` function. This essentially created an open email relay, allowing attackers to send arbitrary spam or phishing emails to any recipient using the application's email servers, bypassing authentication.
**Learning:** Never trust user input for determining destinations of external communications (emails, SMS, webhooks) without proper validation, mapping, or static configuration. In contact forms, the recipient should be hardcoded or retrieved from internal settings, while the user's input should only be used as metadata within the message content.
**Prevention:** Always hardcode or use a trusted configuration value (e.g., `settings.DEFAULT_FROM_EMAIL`) for the recipient address when sending emails from user-submitted forms. Include the user's email address in the message body or as a Reply-To header instead.
